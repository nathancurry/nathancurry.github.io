My girlfriend is out of town, so now’s a great time to set up VLANs.

The Map

I’m switching to 10.3.3.0/24, since that’s easier to type than 192.168.7.0/24, especially in the dark.

Private LAN

  • VLAN 1
  • 10.3.3.0/25
  • Pool 80-126

Guest WiFi

  • VLAN 99
  • 10.3.3.128/26
  • No routing to other subnets

IoT

  • VLAN 5
  • 10.3.3.208/28
  • Restricted routes available to other VLANs

DMZ

  • VLAN 10
  • 10.10.10.0/29
  • Routes assigned as needed to other VLANs
  • This is a proof of concept. My current setup provides approximately zero extra security, since all my DMZ hosts are VMs hosted on non-DMZ hosts.

Management

  • VLAN 20
  • 10.3.3.240/28
  • No routing the outside world

My router has 5 ports. eth0 is the WAN interface, and eth1-eth4 are assigned to switch0. Since bonding is only available in software in the ER-X (which is terribly slow), I’m going to assign VLAN 1 to eth0, and VLANs 5, 10, and 20 to eth1. I’m going to assign VLAN 99 to eth3, and keep eth4 as a recovery configuration port.

Configuring the ER-X

Set up recovery interface

The Edgerouters let you break everything terribly. You should be able to recover from a commit by power cycling if you haven’t saved, but just in case:

nc@gw# delete interfaces switch switch0 switch-port interface eth4
nc@gw# set interfaces ethernet eth4 address 192.168.1.1/24
nc@gw# commit
nc@gw# save

It also helps because you can connect to the management port and not lose connectivity if you do something stupid.

Set up VLANs

EdgeOS won’t route between VLAN sub-interfaces and the base switch0 device, which is not the best documented thing in the world.

nc@gw# delete interfaces switch switch0 address 10.3.3.65/26
nc@gw# set interfaces switch switch0 vif 1 address 10.3.3.65/26
nc@gw# commit
nc@gw# save

Ok, now we can get rolling. I actually did these VLANs one at a time, so I could test:

nc@gw# set interfaces switch switch0 vif 5 address 10.3.3.209/28
nc@gw# set interfaces switch switch0 vif 10 address 10.3.3.1/26
nc@gw# set interfaces switch switch0 vif 20 address 10.3.3.254/28
nc@gw# set interfaces switch switch0 vif 99 address 10.3.3.129/26

# Create temporary VLAN to migrate hosts
nc@gw# set interfaces switch switch0 vif 101 address 192.168.7.1/24

# Configure Inter-VLAN Routing
nc@gw# set interfaces switch switch0 switch-port vlan-aware enable
nc@gw# set port-forward lan-interface switch0.1
nc@gw# set port-forward lan-interface switch0.5
nc@gw# set port-forward lan-interface switch0.10
nc@gw# set port-forward lan-interface switch0.20
nc@gw# set port-forward lan-interface switch0.99
nc@gw# set port-forward lan-interface switch0.101

# Set DHCP for default VLAN
nc@gw# set service dhcp-server shared-network-name Private subnet 10.3.3.64/26 default-router 10.3.3.65
nc@gw# set service dhcp-server shared-network-name Private subnet 10.3.3.64/26 dns-server 10.3.3.65
nc@gw# set service dhcp-server shared-network-name Private subnet 10.3.3.64/26 domain-name lan.nathancurry.com
nc@gw# set service dhcp-server shared-network-name Private subnet 10.3.3.64/26 start 10.3.3.81 stop 10.3.3.127

# [snip] repeat dhcp-server steps for VLAN 99[/snip]

# Set VLAN 99 lease time to 6 hours
nc@gw# set service dhcp-server shared-network-name Guest subnet 10.3.3.128/26 lease 21600

Configure ER-X VLAN trunking

# Uplink 1
nc@gw# set interfaces switch switch0 switch-port interface eth1  vlan pvid 1
# Uplink 2
nc@gw# set interfaces switch switch0 switch-port interface eth2  vlan pvid 5
nc@gw# set interfaces switch switch0 switch-port interface eth2  vlan vid 10
nc@gw# set interfaces switch switch0 switch-port interface eth2  vlan vid 20
# Guest WiFi
nc@gw# set interfaces switch switch0 switch-port interface eth3  vlan pvid 99
# Temporary migration VLAN
nc@gw# set interfaces switch switch0 switch-port interface eth2  vlan vid 101

Configure VLANs on Cisco Switch

# Enable VLANs and management interface
switch(config)#ip default-gateway 10.3.3.254
switch(config)#vlan 10
switch(config-vlan)#int vlan 10
switch(config-if)#description DMZ
switch(config-if)#vlan 20
switch(config-if)#description Management VLAN
switch(config-if)#ip address 10.3.3.251 255.255.255.240
switch(config-vlan)#exit

# Configure links to router
switch(config)#int g0/1
switch(config-if)#description VLAN 1 to gw
switch(config-if)#switchport mode access
switch(config-if)#switchport access vlan 1
switch(config-if)#int g0/2
switch(config-if)#description VLANs 10,20 to gw
switch(config-if)#switchport mode trunk
switch(config-if)#switchport trunk allowed vlan 5,10,20,101
switch(config-if)#switchport trunk native vlan 5

Configure VLAN access

switch(config)#int g0/3
switch(config-if)#description Link to private WAP
switch(config-if)#switchport mode trunk
switch(config-if)#switchport trunk native vlan 1
switch(config-if-range)#switchport trunk allowed vlan 1,5,20
switch(config-if)#int range g0/4-6
switch(config-if-range)#description Hypervisor
switch(config-if)#switchport mode trunk
switch(config-if-range)#switchport trunk native vlan 1
switch(config-if-range)#switchport trunk allowed vlan 1,10,101

Migrating Proxmox cluster

This was the part I was dreading, but it was very easy. First, I log in and shut down my guests (I need to update hardware settings anyway), and then:

# Both hosts
root@gold:~# systemctl stop pve-cluster corosync
root@gold:~# pmxcls -l
root@gold:~# sed -i 's/192.168.7/10.3.3/g/' /etc/network/interfaces /etc/hosts /etc/corosync/corosync.conf
root@gold:~# kill -SIGTERM `pgrep -f pmxcfs`

# One host
root@gold:~# gluster volume stop gluster

And reboot. I had minor issues with the machines syncing on the way back up since one host got stuck on the way down. All I did is follow this section again, and started corosync and pve-cluster services at exactly the same time on both hosts.

Migrating VMs

There are currently no sensitive services on my VMs, so I just logged in via the Proxmox console and updated the network info, and set static bindings.

Cleaning up and securing

On the Edgerouter:

nc@gw# delete port-forward lan-interface switch0.101
nc@gw# delete interfaces switch switch0 vif 101

On the Catalyst switch:

switch(config)#no vlan 101
switch(config)#int g0/2
switch(config-if)#switchport trunk allowed VLAN 10,20,5
switch(config-if)#int range g0/4-6
switch(config-if-range)#switchport trunk allowed vlan 1,10

Firewall Configuration

For now, I’ve got working inter-VLAN routing, and nothing on any of my restricted VLANs. I’m going to work on some other projects and return to this with fresh eyes.
