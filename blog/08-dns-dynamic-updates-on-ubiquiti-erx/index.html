DNS Dynamic Updates With DNSMasq on Ubiquiti ER-X

Posted on 01 Sep 2018 2 mins read

I had previously spun up DNS and DHCP on a CentOS 7 VM, but I’m currently blowing out my entire environment to migrate to oVirt. Plus, I’ve since gotten a Ubiquiti Edgerouter X, which is a mighty capable device.

The default DHCP server is ISC dhcpd, and the configuration doesn’t update DNS entries (or at least not reliably), so I chose to use dnsmasq

Prerequisites

I never turned on the DHCP server, but you’ll have to turn it off. You also need access to the console. You can access it through the WebUI, but it’s a little clunky compared to a good old fashioned SSH session.

The ER-X is debian based, with root password, so log in as your admin user and switch users with sudo su -

Config Files

# /etc/dnsmasq.d/dnsmasq.conf
domain-needed
bogus-priv

# Configure local lan domain
domain=lan.nathancurry.com
expand-hosts
local=/lan.nathancurry.com/

# Set which interfaces to listen on
listen-address=127.0.0.1
listen-address=192.168.7.1
bind-interfaces

# Failover DNS servers
server=75.75.75.75
server=75.75.76.76

# DHCP section
dhcp-range=lan,192.168.7.101,192.168.7.200
dhcp-option=lan,3,192.168.7.1
dhcp-option=lan,6,192.168.7.1

And I kept my static mappings in a separate file, and left myself comments:

#/etc/dnsmasq.d/static.conf
#dhcp-host=mylaptop,192.168.0.199,36h
# OR
#dhcp-host=aa:bb:cc:dd:ee:ff,192.168.111.50

#gold
dhcp-host=18:66:da:14:22:6e,192.168.7.71

#silver
dhcp-host=48:0f:cf:44:1c:d6,192.168.7.72

#tiny
dhcp-host=7c:5c:f8:1a:0d:88,192.168.7.77

Spin it up and test it

root@router:~# service dnsmasq restart
[ ok ] Restarting DNS forwarder and DHCP server: dnsmasq.

And on my laptop:

nc@tiny: ~ $ dig gold
